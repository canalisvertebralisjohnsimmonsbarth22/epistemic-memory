## 🧩 How It Works

epistemic-memory treats memory as something that can change. It does not store a fact as fixed and final. It tracks:

- What the system thinks it knows
- How confident it is
- How old the memory is
- Whether later input conflicts with older input

This helps the app keep memory that is useful, current, and easy to review.

## 📝 Main Parts of the App

### Memory Entries

A memory entry is one piece of information about a person. For example:

- A name
- A preference
- A role
- A habit
- A detail from a past chat

### Confidence

Each memory can have a confidence level. That tells the app how much trust to place in it.

### Decay

Old memory can lose weight over time. This helps the app avoid holding on to stale details for too long.

### Contradictions

If the app sees a new detail that does not match an old one, it can mark the conflict. That helps you see where memory needs review.

## 🛠️ Basic Use
